## 画面遷移

```
[S01 ダッシュボード]
  ├─→ [S06 見積書一覧]
  ├─→ [S11 請求書一覧]
  └─→ [S02 顧客一覧]

[S02 顧客一覧]
  ├─→ [S03 顧客登録]
  └─→ [S04 顧客詳細]
        └─→ [S05 顧客編集]

[S06 見積書一覧]
  ├─→ [S07 見積書作成]
  └─→ [S08 見積書詳細]
        ├─→ [S09 見積書編集]    ※ draft のみ
        ├─→ [S10 見積書PDF]
        └─→ [S12 請求書詳細]   ※ converted 後、変換先へリンク

[S07 見積書作成]
  └─→ 保存後 [S08 見積書詳細]

[S11 請求書一覧]
  └─→ [S12 請求書詳細]
        ├─→ [S13 請求書編集]   ※ draft のみ
        ├─→ [S14 請求書PDF]
        └─→ [S08 見積書詳細]   ※ 変換元へリンク
```

## 見積書ステータス遷移

```
[draft] ──(送付済みにする)──→ [sent]
   │                             │
  (削除)                    (承認済みにする)──→ [accepted]──(請求書に変換)──→ [converted]
                                 │
                            (却下済みにする)──→ [rejected]
```

| 遷移 | トリガー | 条件 |
|------|---------|------|
| `draft` → `sent` | 詳細画面の「送付済みにする」ボタン | 明細行が1件以上ある |
| `sent` → `accepted` | 「承認済みにする」ボタン | — |
| `sent` → `rejected` | 「却下済みにする」ボタン | — |
| `accepted` → `converted` | 「請求書に変換」ボタン | — |
| `draft` → 削除 | 「削除」ボタン | `draft` のみ削除可 |

**変換時の動作：**
1. `Quote.status` を `converted` にセット
2. `Quote` の明細行・顧客・金額をコピーして `Invoice`（status=`draft`）を新規作成
3. `Invoice.quote_id` に元の `Quote.id` を記録
4. 詳細画面を作成した `Invoice` にリダイレクト

## 請求書ステータス遷移

```
[draft] ──(送付済みにする)──→ [sent]
                                  │
                             (入金済みにする)──→ [paid]
                                  │
                             (期日超過にする)──→ [overdue]──(入金済みにする)──→ [paid]
```

| 遷移 | トリガー | 条件 |
|------|---------|------|
| `draft` → `sent` | 「送付済みにする」ボタン | 明細行が1件以上ある |
| `sent` → `paid` | 「入金済みにする」ボタン | — |
| `sent` → `overdue` | 「期日超過にする」ボタン（手動） | — |
| `overdue` → `paid` | 「入金済みにする」ボタン | — |

> MVP では期日超過の自動検出は行わない。手動でステータスを変更する。

---

## ダッシュボード表示項目

| セクション | 内容 |
|-----------|------|
| 未対応の見積書 | `sent` 状態の件数・金額合計 |
| 入金待ちの請求書 | `sent` + `overdue` の件数・金額合計 |
| 最近の見積書 | 直近5件（ステータス・顧客名・金額） |
| 最近の請求書 | 直近5件（ステータス・顧客名・金額） |

---

## 不正遷移のハンドリング

- 許可されていない遷移を試みた場合 → `400 Bad Request`、エラーメッセージをフラッシュ表示
- `converted` の見積書の編集・削除 → `403 Forbidden`
- `paid` の請求書の編集 → `403 Forbidden`

---

## MVP スコープ外

- **請求書の手動作成：** 請求書は見積書からの変換（`accepted` → `converted`）のみで生成する。`/invoices/new` 画面は存在しない。`invoices.quote_id` の NULL 許容は将来の手動作成機能のために確保。

---

## 顧客削除ルール

- 見積書または請求書が1件以上紐づいている顧客は削除不可（`400 Bad Request`）
- 削除前にすべての関連ドキュメントを削除・移管してから顧客を削除する
